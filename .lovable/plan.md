# План переезда на Timeweb Cloud (РФ)

## Что важно понять с самого начала

Сейчас сайт работает на инфраструктуре Lovable:
- **Хостинг приложения** — Cloudflare Workers (международный CDN)
- **База данных, авторизация, файловое хранилище** — Supabase (серверы в ЕС)
- **AI для примерки** — Lovable AI Gateway (Google Gemini)
- **Email** — Resend (через коннектор Lovable)

Чтобы данные клиентов (заявки, фото для примерки) хранились в РФ по 152-ФЗ, нужно **полностью уйти от Supabase и Cloudflare**, оставив только Resend (для писем) и заменив Lovable AI на OpenRouter. Я подготовлю код, но **сам переезд и настройку сервера нужно будет выполнить вручную** — Lovable не управляет внешними хостингами.

---

## Этап 1. Подготовка инфраструктуры на Timeweb (вы делаете сами)

### 1.1 Заказать тариф
Берём **Timeweb Cloud → VPS / Cloud-сервер** (НЕ обычный shared-хостинг — он не подойдёт для Node.js).

Минимальная конфигурация для старта:
- 2 vCPU, 2 ГБ RAM, 30 ГБ NVMe, Ubuntu 22.04
- ~300–500 ₽/мес
- Локация: **Москва** или **Санкт-Петербург** (для 152-ФЗ)

### 1.2 Дополнительные сервисы Timeweb
- **Managed PostgreSQL** (~400 ₽/мес) — замена Supabase Database
- **S3-хранилище** (~5 ₽/ГБ) — замена Supabase Storage для фото примерки

### 1.3 Зарегистрировать аккаунты
- **OpenRouter** (openrouter.ai) — пополнить на $5–10, получить API-ключ
- **Resend** — у вас уже есть через Lovable, нужно будет получить отдельный API-ключ напрямую на resend.com
- Российская карта Мир / Альфа для оплаты Timeweb

## Этап 2. Доработки кода в Lovable (это делаю я)

Перед экспортом нужно «отвязать» проект от Lovable-специфичных вещей.

### 2.1 Заменить Lovable AI на OpenRouter
Файл `src/lib/tryon.functions.ts` сейчас вызывает `ai.gateway.lovable.dev`. Перепишу на OpenRouter (`openrouter.ai/api/v1/chat/completions`), модель — `google/gemini-2.5-flash-image-preview` (она же делает примерку). Переменная окружения: `OPENROUTER_API_KEY`.

### 2.2 Заменить Lovable Email Gateway на прямой Resend
Файл `src/routes/api/public/order.ts` сейчас идёт через `connector-gateway.lovable.dev/resend`. Перепишу на прямой вызов `api.resend.com/emails` с собственным `RESEND_API_KEY`.

### 2.3 Заменить Cloudflare Workers на Node.js-адаптер
Сейчас в `wrangler.jsonc` и `vite.config.ts` стоит сборка под Cloudflare. Нужно перевести TanStack Start на **Node.js preset** — это штатный режим фреймворка, приложение собирается в обычный `node server.js`.

### 2.4 Подготовить SQL-дамп схемы
Экспортирую структуру таблиц + storage-bucket `tryon` в один SQL-файл — его можно будет залить в новый PostgreSQL на Timeweb. Заявок в БД сейчас нет (`lead_requests` уже не используется — заявки идут только письмами), так что данные переносить не нужно.

### 2.5 Заменить Supabase Auth и Storage
Сейчас Supabase используется для:
- **Storage `tryon`** — загрузка фото пользователя перед AI-примеркой
- **Auth** — настроено, но фактически на лендинге не используется (страница `/auth` есть, но кнопок входа на сайте нет)

Заменю Storage на **S3-совместимое хранилище Timeweb** (через AWS SDK). Auth выпилю полностью, раз не используется — это сильно упростит переезд.

## Этап 3. Развёртывание на Timeweb (вы + я с инструкциями)

После всех правок я дам пошаговую инструкцию:
1. Скачать код проекта из Lovable (кнопка GitHub → Export)
2. Залить на сервер Timeweb по SSH
3. Установить Node.js 20, pm2, Nginx
4. Прописать переменные окружения (`.env`): `DATABASE_URL`, `OPENROUTER_API_KEY`, `RESEND_API_KEY`, S3-ключи
5. Запустить `npm install && npm run build && pm2 start server.js`
6. Настроить Nginx как reverse-proxy + Let's Encrypt SSL
7. Залить SQL-дамп в PostgreSQL

---

## Этап 4. Подключение домена

Когда сайт заработает на IP-адресе Timeweb:
1. Купить домен (можно в Timeweb или у reg.ru)
2. Прописать A-запись на IP сервера
3. Получить SSL-сертификат через Let's Encrypt (бесплатно, автообновление)
